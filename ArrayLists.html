ArrayLists


ArrayLists, or Lists, can hold objects and only objects. This means that they cannot hold primitives such as ints, strings, or boolean. If we wanted to fill our List with these sorts of primitives, we would have to use wrapper classes which are classes that encapsulate normal primitives to make them usable. The wrapper object for int is Integer, string is String, and boolean is Boolean. They are simple in that all you have to do is capitalize the first word to wrap them. ArrayLists are mutable, which means that you can add as many elements as you want. The List adjusts its size by adding twice as many slots when the user adds an object that exceeds the previous carrying capacity, as shown below.

Intialization

Remember to import java.util

The ArrayList can be initialized in multiple ways, but they are all very similar to each other. Here are most of them.


ArrayList < Object > list = new ArrayList< Object >();

List < Object > list = new ArrayList< Object >();

ArrayList < Object > list = new ArrayList<>();

List < Object > list = new ArrayList<>();


On these initializations, you will notice some common things amongst them all. On the left of the equal sign, we must always put the object type in the <>, so in this case, it was Integer. We do this so the computer knows what type of object to put in the List .We also must put our object's name right next to the <>, which in this case was list. We must also make sure to put the 'new ArrayList' right after the equal sign, so the computer knows to make a new ArrayList. Finally, we must put the parenthesis at the end and a semicolon, as this calls to the class's constructor method. Now let's talk about the optional parts of the initialization. The first word in the initialization can be formatted as either List or ArrayList. The computer notices both of them as ArrayList, as long as the word after the keyword 'new' is ArrayList. Also, it is optional to rewrite the object type that you are putting in the <> after the equal sign. Doing this sets up extra security parameters and ensures that the method does not get corrupted, but is by no means is this necessary. You can also have a starting size for your ArrayList by putting the desired size in the parenthesis, like this.


ArrayList < Object > list = new ArrayList< Object >(5);


You might ask, why is this necessary, as ArrayLists are immutable? Having a predetermined size makes the code run faster and makes the overall experience less cumbersome for the computer.

Useful methods

.add(Object x) = adds Object x at the end of the list

.add(Object x, int y) = adds Object x at index y.

.get(int x) = returns the object at index x.

.indexOf(Object x) = returns index of Object x.

.remove(int x) = removes and returns Object at index x.

.remove(Object o) = removes first instance of Object o.

.contains(Object o) = returns true if list contains specified Object.

.size() = returns the length of the ArrayList

Iteration

ArrayLists can be iterated through like a normal Array, with a for or for-each loop.


for(int i= 0; i < list.size(); i++)

{

//code

}


The noticeable difference here from the iteration of a normal Array is that we use .size() instead of .length. This is because ArrayList is a different class from the Array class. In the Array class, .length returns the instance variable that holds the length, and in the ArrayList class, .size() is a call to a method that returns the length of the ArrayList.
