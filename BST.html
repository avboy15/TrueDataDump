Binary Search Tree


The binary search tree(BST) is one of the most simple ADTs to visualize yet one of the most difficult to program. Basically, the BST starts with a 'root' Node that has two instance variables that connect it to two other Nodes. The first Node is the left Node, and it contains an object that is less in value than the root Node. The second Node is the right Node, and it contains an object that is more in value than the root Node. Each of these right and left Nodes have left and right Nodes, and so on. This means that we would find the lowest value at the leftmost end of the tree, and we would find the highest value at the rightmost end of the tree. For non-integer objects, we would use the compareTo() method to see which value is higher and lower.

Initialization

Since there is technically no Binary Search Tree in the Java Collections class (There is TreeMap and TreeSets, but these are functionally similar to HashMaps), I will just show you the Node class in this section. Like the LinkedList class, we would find the Node class for the BST inside the BST class itself.


Private class TreeNode

{

TreeNode Left;

TreeNode Right;

int Val;

Private TreeNode(int num)

{

Val = num;

Left = null;

Right = null;

}

}


Like I said before, the Node has a left Node, right Node, and a value itself. When we make the Node, the left and right Nodes are set to null and the value is decided by the user via the parameter. If you are confused about the left and right Nodes, check out 'Introduction to Abstract Data Types'.

Iteration

Trees are most easily iterated through with a technique called recursive backtracking. This is basically using two methods , one of which is recursive, to do the work of a single method. Let's look at the class and the add method for a BST to see what I am talking about.


Public class BST

{

TreeNode root;

Private class TreeNode

{

//code for TreeNode

}

Public BST(int num)

{

root = new TreeNode(num);

}

Public void add(int index)

{

addHelper(root,num);

}

Public void addHelper(TreeNode node, int num)

{

if(node == null)

{

node = new TreeNode(num);

}

else if(num < node.val)

{

addHelper(node.left, num);

}

else

{

addHelper(node.right, num);

}

}

}


Once we initialized the root, we can see two methods, one called add and the other called addHelper. In the regular add method, we have the number that we want to add as a parameter. However, the only code in the add method is a call to the addHelper method, with its parameters as the root node and add's parameter. In the addHelper method, we use recursion, meaning we call to the method inside the same method. The first if statement checks if the node is null, and if it is, then it changes the null node to a new TreeNode with the num as a value. The else if statement checks if the num is less than the value of the current node. If so, then it calls to the addHelper method, except this time, the node parameter is node.left instead of root. The last else statement does this same thing to the right node. The recursive calls in this method keep updating the value of the node either to the right or to the left node, depending on the value of nums and the value of the existing nodes in the tree. Once the method encounters a null node, it makes a new node of value num at that null reference. This null reference, determined by the else if and else statements, is the next value in the tree. This is a visualization of a BST.
