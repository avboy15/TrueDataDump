The HashMap is a very unique dataset, in that it does not call or iterate through its value in numeric order. Instead, it uses 'keys' to call a 'value'. A key is an object that the computer pairs with a certain value, so when we call the key, it returns a value. Like an ArrayList, it can only store objects in the key and value spots, so you may have to use some wrapper classes. HashMaps are also mutable, but can be initialized with a starting size to save time and space in the computer. If you wanted to create a HashMap that doesn't have repeats, then make a HashSet instead. Here is a visualization of the HashMap.

So if we call the 'three' key, the program will return the value indexed at that key, which in this case is 3

Intialization

Remember to import java.util

The initialization for the HashMap is very similar to the initialization for the ArrayList.


HashMap < String,Integer > map = new HashMap< String,Integer >();

HashMap < String,Integer > map = new HashMap<>();

Map < String,Integer > map = new HashMap< String,Integer >();

Map < String,Integer > map = new HashMap<>();

HashMap < String,Integer > map = new HashMap<>(5);


As you can see, the HashMap is different in that instead of having a single data type in the <>, it has two. The first data type is the key, which in this case is a string, and the seconds data type is the value, which in this case is an Integer. Of course, you can make the key and value any combination of object type that you want, so don't feel limited to your possibilities.

Useful methods

.get(Object key) - Key is a parameter, returns the value of the key.

.put(K key, V value) - create a new key and value to place in your HashMap.

.remove(Object key) - removes the key and the value , but also returns the value

.containsKey(Object key) - checks if the HashMap contains this key.

.containsValue(Object value) - checks if the HashMap contains this value.

.size() - returns size of the HashMap.

.getSet() - returns an iterable collection of keys in the hashmap

.values() - returns an iterable collection of values in the hashmap

Iteration

Iteration through HashMaps are a little more complex than what we have covered so far, since we have to worry about both keys and values. We can iterate using three methods. First, we can iterate through the keys only.


for(String key: list.getSet())

{

//code

}


As mentioned in the 'useful methods' section, the 'getSet()' method returns an iterable collection of all of the sets in the code. We can also iterate through the values exclusively, like this.


for(int value: list.values())

{

//code

}


Like the 'getSet()' method before, the 'values()' method returns an iterable collection of all the values in the code.

The hashmap is not really intended for iteration. Instead, it's main purpose is to quickly call a value with a key. This is why iterating through hashmaps is complicated and cumbersome.
