Stacks


Our first ADT is the Stack. The Stack is almost exactly what it sounds like. Imagine placing a deck of cards face down on a table, one by one. The first card that you put on the table will be the bottom of the stack. If you want to remove that card from the stack or see what it's value is, then you would have to take out every card on top of it one by one. Therefore, the first value added on the stack is always going to be the last one out (FILO). That being said, the last value put on the stack will be the first one out. Stacks are used in most internet browser, when you go to the previous page.Stacks are mutable.

You cannot initialize a stack with a set number of starting slots because it is an Abstract Data Type.

Useful Methods

The methods for Stack are very different from the methods that we have already seen, so it is an integral that you get a good understanding of them.

.push(Object o) - This adds the Object o on the top of the stack.

.pop() - This removes the top Object on the stack and returns it's value.

.peek() - This returns the value of the top Object on the stack and does not alter the stack.

.empty() - Checks if stack is empty.

.search(Object o) - returns position of the Object o.

Iteration

Iterating through Stacks is weird, since you cannot return a value unless it is on the top of the stack. Using the new methods we learned in the previous page (have link to useful methods), we can iterate through a stack with easy.


whlie(myStack.empty()==false)

{

myStack.pop();

The while loop uses the .empty() method to check if the stack has elements in it. While the stack has elements, the .pop() methods returns the top element and gets rid of that element right after. When working with a Stack, be careful, as the only way to get to the next element is to pop the top element off. Alternatively, we could just peek at the top element and remove it later. Like this.


while(myStack.empty()==false)

{

myStack.peek();

//code

myStack.pop();

}
